# 페르소나 시스템 사용 가이드

> **버전**: 2.0.0
> **v2 핵심 변경**: 증거 기반 완료 체계, 기능별 검증 사이클, 실시간 진도 추적

## 설치 방법

### 1. Claude Code에서 프로젝트 초기화
```bash
/init
```

### 2. 페르소나 파일 복사
생성된 `.claude/` 디렉토리에 페르소나 파일들을 복사:

```
.claude/
├── CLAUDE.md           # 메인 설정 파일 내용 복사
└── personas/           # 디렉토리 생성 후 각 팀 파일 복사
    ├── pm팀.md
    ├── 리서치팀.md
    ├── 기획설계팀.md
    ├── 디자인팀.md
    ├── 개발1팀.md
    ├── 개발2팀.md
    ├── 개발3팀.md
    ├── qa.md
    ├── monitoring.md
    └── 시뮬레이션.md
```

### 3. 프로젝트 초기화 실행
```bash
@PM팀 프로젝트 초기화
```

## 기본 사용법

### 방법 A: 기획문서가 있을 때
```bash
# 기획문서를 첨부하고 개발 요청
@PM팀 기획문서 기반 개발 요청
# [기획문서/PRD 첨부]

# 프로세스:
# ① 문서 분석 → ② 설계 → ③ 기능별 개발 → ④ 기능별 실행 검증
# → ⑤ 누적 통합 검증 → ⑥ QA/시뮬레이션 → ⑦ 최종 인수
```

### 방법 B: 아이디어를 바로 말할 때
```bash
# 만들고 싶은 것을 자유롭게 설명
@PM팀 즉석 개발 요청
# "관리자가 회원 목록을 보고 수정할 수 있는 페이지"

# 프로세스:
# ⓪ 요청 해석 + 사용자 확인 → ⓪-2 미니설계서 → ① 기능별 개발
# → ② 기능별 실행 검증 → ③ 누적 통합 검증 → ④ QA → ⑤ 최종 확인
```

### 진행 상황 확인
```bash
@PM팀 현재 진행 현황 보고
```

## v2 핵심 규칙 (이전 버전과의 차이)

### 1. 기능 1개 = 1사이클 (가장 중요한 변경)

```
❌ v1 (기존): 모든 기능 개발 → 한번에 테스트 → 오류 폭탄
✅ v2 (변경): 기능1 개발→검증→통과 → 기능2 개발→검증→통합확인 → ...
```

### 2. 증거 기반 완료

"코드 작성 완료"는 완료가 아닙니다. 아래가 모두 충족되어야 "완료":
- 코드를 실제로 실행함
- 에러가 0건임
- 기존 기능과 통합 실행해서 깨진 것 없음
- 화면이 있으면 화면 접속 확인함

### 3. 실시간 진도 추적

기능 1개 완료 시마다 진행 현황 보고가 자동 출력됩니다:
```
📊 개발 진행 현황
[전체 진도] ████░░░░░░ 40% (2/5 기능)
| 기능 | 상태 | 담당 | 검증 결과 |
...
```

### 4. 페르소나 참여 추적

최종 완료 시 어떤 페르소나가 얼마나 참여했는지 리포트됩니다.

## 팀 호출 방법

### PM팀 경유 (권장)
```bash
@PM팀 [요청사항]
```
PM팀이 적절한 팀/담당자에게 자동 분배

### 특정 팀 직접 호출
```bash
@개발1팀 [개발 요청]
@QA팀 [테스트 요청]
@디자인팀 [디자인 요청]
```

### 특정 담당자 호출
```bash
@PM팀.유진호 [소통관 요청]
@개발1팀.김태현 [백엔드 요청]
@QA팀.김정훈 [QA 리드 요청]
```

## 주요 담당자 Quick Reference

### PM팀 소통관 (요청 접수)
| 담당자 | 담당 영역 |
|--------|----------|
| 유진호 | 복합/대규모 요청, 신규 프로젝트, **★ 요청 해석** |
| 서민지 | 기술 요청, 개발팀 분배 |
| 임채원 | 디자인/QA 요청 |

### 일관성 PM (검토)
| 담당자 | 검토 영역 |
|--------|----------|
| 정대훈 | 코드 컨벤션, 네이밍 |
| 한소라 | 디자인 토큰, 컴포넌트 |
| 강현우 | UI/UX, 접근성 |
| 윤성호 | DB 스키마 |
| 배지영 | 쿼리 성능 |

### ★ 검증 게이트 관리
| 담당자 | 역할 |
|--------|------|
| 박준혁 (품질PM) | 검증 게이트 총괄, 증거 확인 |
| 강민호 (PM 리더) | 최종 인수 판정 |

### 개발팀 리드
| 담당자 | 팀 | 전문 |
|--------|-----|------|
| 김태현 | 개발1팀 | 백엔드/API |
| 신예진 | 개발1팀 | 프론트엔드 |
| 이준혁 | 개발2팀 | 레거시/마이그레이션 |
| 장우혁 | 개발3팀 | Skill 개발, **★ 검증 자동화** |

## 트러블슈팅

### 페르소나가 응답하지 않는 경우
1. CLAUDE.md 파일이 올바른 위치에 있는지 확인
2. personas/ 디렉토리 구조 확인
3. 파일 인코딩이 UTF-8인지 확인

### 팀 분배가 이상한 경우
```bash
@PM팀 현재 팀 배정 상태 확인
```

### ★ 검증이 계속 실패하는 경우
```bash
@PM팀 [기능명] 검증 실패 원인 분석 요청
```

### 프로세스 초기화
```bash
@PM팀 프로젝트 초기화
```

## 주의사항

1. **모든 요청은 `@PM팀`으로 시작** - 직접 개발 금지
2. **팀 간 직접 소통 금지** - PM팀 경유 필수
3. **PRD 필수** - 없으면 미니설계서 필수 (즉석 요청 시 자동 작성)
4. **일관성 검토 필수** - 코드/디자인/DB 변경 시
5. **★ 실행 검증 없이 "완료" 보고 금지**
6. **★ 여러 기능 한번에 개발 후 몰아서 테스트 금지**
7. **★ 통합 검증 생략 금지**
