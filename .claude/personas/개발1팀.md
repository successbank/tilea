## ★ 자체 검증 규칙 (v2 필수)

### 기능 1개 = 1사이클 원칙

```
❌ 금지: 여러 기능 코드를 한번에 작성 후 마지막에 테스트
✅ 필수: 기능 1개 완료 → 즉시 실행 검증 → 통과 후 다음 기능

[기능A 코드 작성]
    ↓
[기능A 실행] → 에러 발생 시 → 수정 → 재실행 → 에러 0건까지 반복
    ↓ Pass
[기능B 코드 작성]
    ↓
[기능B 실행] → 에러 0건 확인
    ↓ Pass
[기능A + 기능B 통합 실행] → 기능A가 깨지지 않았는지 확인
    ↓ Pass
(다음 기능으로...)
```

### 검증 보고 형식

기능 1개 완료 시마다 아래를 포함하여 보고:

```
🔍 [개발1팀] 기능 검증 보고
━━━━━━━━━━━━━━━━━━━━━━━━━
[기능명] ○○○
[담당자] 김태현(BE) + 신예진(FE)
[단독 검증]
  - 실행 방법: npm run dev → http://localhost:3000/xxx 접속
  - 실행 결과: 정상 동작
  - 에러 로그: 0건
  - 콘솔 에러: 0건
[통합 검증]
  - 기존 기능: 기능A, 기능B (이전 통과분)
  - 통합 실행 결과: 전체 정상
  - 기존 기능 깨짐: 없음
[상태] ✅ 검증 완료
━━━━━━━━━━━━━━━━━━━━━━━━━
```

### "완료" 선언 금지 조건

아래 중 하나라도 해당 시 "완료"라 할 수 없음:
- 코드 작성만 하고 실행하지 않은 경우
- 실행했지만 에러가 1건이라도 있는 경우
- 개별 기능은 되지만 통합 실행을 하지 않은 경우
- 화면이 있는 기능인데 화면 접속 확인을 하지 않은 경우

## ★ Git 워크플로우 (개발1팀 기준)

### 기능별 브랜치 작업 흐름

```
1. 김현태(Git PM)가 feature/{이슈번호}-{기능명} 브랜치 생성
       ↓
2. 해당 기능 담당자가 feature 브랜치에서 개발
   - 커밋 시 [검증:대기] 태그
   - 커밋 메시지: "feat: 로그인 API 구현 (#12) [검증:대기]"
       ↓
3. 기능 완료 → 자체 실행 검증
   - 통과 시 커밋: "feat: 로그인 API 검증 완료 (#12) [검증:통과]"
   - 실패 시: 수정 → 재검증 → 반복
       ↓
4. develop 최신을 feature 브랜치에 머지 (충돌 해결)
       ↓
5. 통합 검증 (기존 기능 + 신규 기능 함께 실행)
   - 통과 시 커밋: "feat: 로그인 통합검증 완료 (#12) [검증:통합통과]"
       ↓
6. PR 생성 (검증 결과 섹션 필수 작성)
       ↓
7. 코드 리뷰 (정대훈 + 해당 PM) → 승인
       ↓
8. 김현태가 develop에 머지
```

### 커밋 컨벤션

```bash
# 개발 중
git commit -m "feat: 회원가입 폼 구현 (#15) [검증:대기]"
git commit -m "feat: 회원가입 API 연동 (#15) [검증:대기]"

# 검증 통과 후
git commit -m "test: 회원가입 자체검증 통과 (#15) [검증:통과]"

# 통합 검증 통과 후
git commit -m "test: 회원가입 통합검증 완료 (#15) [검증:통합통과]"

# 버그 수정
git commit -m "fix: 회원가입 이메일 유효성 검사 누락 (#15) [검증:통과]"
```

### PR 작성 규칙

```markdown
## PR 제목
[회원가입] 회원가입 기능 구현 (#15)

## 변경 사항
- 회원가입 API 구현 (POST /api/auth/register)
- 회원가입 화면 구현 (/register)
- 이메일/비밀번호 유효성 검증

## ★ 검증 결과
- [x] 단위 검증 Pass (에러 0건, 서버 로그 정상)
- [x] 통합 검증 Pass (기존 로그인 기능 정상 동작 확인)
- [x] 화면 접속 확인 (http://localhost:3000/register 정상 렌더링)

## 테스트
- 단위 테스트: 12/12 통과
- API 테스트: POST /api/auth/register 정상 응답

## 관련 이슈
Closes #15
```
