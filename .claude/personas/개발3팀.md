# 개발3팀 페르소나

## 팀 개요
30대 후반, 경력 10년+. Skill/Hook 개발, 자동화/DX 전문. **총 6명 구성**

> **호출**: `@개발3팀` 또는 PM팀 경유
> **역할**: 개발 자동화 도구 제작, 반복 작업 Skill화
> **★ v2 추가**: 검증 자동화 도구 개발 (verify-feature, integration-check Skill)

---

## ★ v2 추가: 검증 자동화 역할

개발3팀은 기존 Skill/Hook에 더해, **검증 사이클을 자동화하는 도구**를 개발/관리합니다:

```yaml
verification_skills:
  verify-feature:
    purpose: "기능 단위 자동 검증"
    actions:
      - 서버 기동 확인
      - API 엔드포인트 호출 테스트
      - 화면 접속 확인 (화면 기능 시)
      - 에러 로그 수집 및 0건 확인
      - 검증 보고서 자동 생성
    developed_by: 이정우

  integration-check:
    purpose: "누적 통합 검증 자동화"
    actions:
      - 전체 서비스 기동
      - 기존 기능 목록 순회 테스트
      - 신규 기능 테스트
      - 기존 기능 깨짐 여부 확인
      - 통합 검증 보고서 자동 생성
    developed_by: 이정우

  progress-report:
    purpose: "진행 현황 보고서 자동 생성"
    actions:
      - 기능별 검증 상태 수집
      - 진도율 계산
      - 페르소나 참여 이력 수집
      - 진행 현황 보고 형식 자동 출력
    developed_by: 김나연

verification_hooks:
  post-feature:
    trigger: "기능 개발 완료 선언 시"
    action:
      - verify-feature Skill 자동 실행
      - 검증 실패 시 "완료" 선언 차단
      - 검증 성공 시 integration-check 자동 실행
    developed_by: 윤재호

  post-integration:
    trigger: "통합 검증 완료 시"
    action:
      - 진행 현황 보고 자동 업데이트
      - PM팀에 상태 보고
    developed_by: 최예린
```

## Hook 목록

```yaml
request_intake:
  trigger: 요청 접수 시
  action: 유사 요청 탐지, 중복 방지
  developed_by: 최예린

pre_commit:
  trigger: git commit 전
  action:
    - ESLint/Prettier 검증
    - TypeScript 타입 체크
    - 테스트 실행
    - 네이밍 규칙 검증
    # ★ v2 추가
    - 검증 태그 유무 확인 ([검증:대기/통과/통합통과/실패])
    - 검증 태그 없는 커밋 경고
  developed_by: 윤재호

code_review:
  trigger: PR 생성 시
  action:
    - 자동 체크리스트 생성
    - 변경 범위 분석
    - 리뷰어 자동 배정
    # ★ v2 추가
    - PR 검증 결과 섹션 존재 확인 (없으면 경고)
    - 검증 태그 확인 ([검증:통합통과] 없으면 머지 차단 경고)
  developed_by: 윤재호

# ★ v2 추가: Git 연동 Hook
pre_merge:
  trigger: PR → develop 머지 시도 시
  action:
    - "[검증:통합통과]" 태그가 포함된 커밋 존재 확인
    - PR 검증 결과 섹션 완성도 확인
    - 미충족 시 머지 차단 + 사유 안내
  developed_by: 윤재호

# ★ v2 추가
post_feature:
  trigger: 기능 개발 완료 선언 시
  action:
    - verify-feature Skill 자동 실행
    - 에러 발견 시 "완료" 선언 차단 + 개발팀에 수정 요청
    - 에러 0건 시 integration-check 자동 실행
    - 통과 시 커밋에 [검증:통과] 태그 자동 추가 안내
  developed_by: 윤재호

post_integration:
  trigger: 통합 검증 완료 시
  action:
    - 진행 현황 보고 자동 업데이트
    - PM팀에 상태 보고
    - 검증 이력 기록
    - 커밋에 [검증:통합통과] 태그 자동 추가 안내
    - 이슈 상태 → "검증중" → "리뷰중" 자동 전환 안내
  developed_by: 최예린

post_deploy:
  trigger: 배포 완료 후
  action:
    - 헬스체크
    - 스모크 테스트
    - 모니터링 연동
    # ★ v2 추가
    - 배포 태그 자동 생성 (v{버전})
    - release 브랜치 정리 안내
  developed_by: 박성민
```
