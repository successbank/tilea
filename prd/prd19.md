## 19.1 테스트 전략 총괄

### 테스트 피라미드

```
        ┌─────────┐
        │  E2E    │  ← 10% (핵심 플로우만)
        ├─────────┤
        │ 통합    │  ← 30% (API + DB)
        ├─────────┤
        │ 단위    │  ← 60% (함수, 컴포넌트)
        └─────────┘
```

### 도구

| 영역 | 도구 | 용도 |
|------|------|------|
| 단위 테스트 | Jest + React Testing Library | 함수, 컴포넌트 |
| API 테스트 | Supertest | API Route 테스트 |
| E2E 테스트 | Playwright | 브라우저 자동화 |
| 성능 테스트 | k6 | 부하 테스트 |
| 보안 테스트 | OWASP ZAP | 취약점 스캔 |

## 19.4 ★ 사용자 검증 가이드

> **이 섹션은 개발 완료 후 사용자(발주자)가 직접 기능을 검증하는 방법입니다.**

### 검증 환경 준비

```
1. https://tilea.kr 접속
2. 테스트 계정 로그인 (관리자 제공)
   - 일반 회원: test@tilea.kr / Test1234!
   - 사업자 회원: business@tilea.kr / Test1234!
   - 관리자: admin@tilea.kr / (별도 제공)
3. 크롬 브라우저 권장 (개발자 도구 F12 → Console 탭 확인)
```

### 검증 시나리오 1: 회원 가입 → 사업자 인증

```
[검증 절차]
1. tilea.kr/auth/register 접속
2. 이메일/비밀번호/이름/전화번호 입력 → 가입 클릭
3. ✅ 확인: 이메일 인증 메일 수신됨
4. 이메일 인증 링크 클릭
5. ✅ 확인: 로그인 성공 → 대시보드 이동
6. /auth/verify-business 접속
7. 사업자등록번호 입력 (테스트: 1234567890)
8. ✅ 확인: "확인됨" 또는 "유효하지 않음" 표시
9. 관리자 계정으로 /admin/users → 승인
10. ✅ 확인: 사업자 메뉴 접근 가능 (재단 계산기, 견적서 등)

[Pass 기준]
- 가입 → 인증 → 로그인 전체 흐름 5분 이내
- 에러 메시지 없이 완료
- 사업자 인증 후 역할 변경 확인
```

### 검증 시나리오 2: 재단 계산기

```
[검증 절차]
1. /dashboard/cutting-optimizer 접속
2. 원판 규격: 2440 × 1220 입력
3. 부재 추가:
   - "상판" 600×400 × 2개
   - "측판" 400×300 × 4개
   - "선반" 580×300 × 3개
4. "최적화 계산" 클릭
5. ✅ 확인: 배치도 SVG 표시 (각 부재 색상 구분, 이름 표시)
6. ✅ 확인: 자투리율(%) 표시
7. ✅ 확인: 필요 원판 수 표시
8. "PDF 다운로드" 클릭
9. ✅ 확인: PDF 열기 → 인쇄 가능한 재단 지시서
10. Kerf를 4mm로 변경 → 재계산
11. ✅ 확인: 배치가 변경됨 (간격 넓어짐)
12. 결 방향 ON → 재계산
13. ✅ 확인: 일부 부재 회전 안됨

[Pass 기준]
- 배치도가 3초 이내 생성
- 부재가 원판 밖으로 나가지 않음
- 자투리율이 합리적 (15~30%)
- PDF에 모든 정보 포함
```

### 검증 시나리오 3: 견적서 생성 → PDF → 전송

```
[검증 절차]
1. /dashboard/estimates 접속 → "새 견적서" 클릭
2. 고객 정보 입력 (이름, 전화, 이메일)
3. 항목 추가:
   - 재료비: 합판 2440×1220 × 3장 × ₩35,000
   - 인건비: 재단 가공 × 1식 × ₩50,000
   - 부자재: 경첩 4개 × ₩3,000
4. ✅ 확인: 소계 자동 계산 (₩167,000)
5. 마진율 15% 입력
6. ✅ 확인: 최종 금액 ₩192,050 표시
7. "PDF 다운로드" 클릭
8. ✅ 확인: 견적서 양식 (사업자 정보, 항목, 금액)
9. "이메일 전송" 클릭
10. ✅ 확인: 고객 이메일에 견적서 PDF 수신

[Pass 기준]
- 금액 계산 정확
- PDF 인쇄 레이아웃 깔끔
- 이메일 정상 수신
```
